# Contents 页面前方数字重复 — 修复说明

## 现象

Contents（目录）页面前面的编号出现重复（例如同一节出现两个 "1." 或页码重复）。

## 原因

目录页在 **Word 模板**（`Gold_Report_Template.docx`）中为静态排版内容，不是通过占位符生成。若模板里目录使用了多套编号列表，或复制粘贴导致编号域重复，就会出现重复数字。

## 修复方式（在 Word 中操作）

1. **打开模板**  
   用 Microsoft Word 打开 `Gold_Report_Template.docx`。

2. **定位目录页**  
   找到 Contents / 目录 所在页。

3. **检查编号来源**  
   - 若为 **手动输入的编号**：删除重复的那一组编号，只保留一套（例如只保留左侧一列 1、2、3…）。  
   - 若为 **“编号”列表**：选中目录段落 → 「开始」→「编号」→ 只保留一种编号格式，或先设为“无”，再重新应用一次编号。  
   - 若为 **域（如 TOC）**：右键目录 →「更新域」→ 选择“更新整个目录”；若仍重复，检查是否插入了两个 TOC 域，删除多余的一个。

4. **保存模板**  
   保存后重新生成报告，确认目录页编号不再重复。

## 代码侧说明

当前 **不** 通过占位符或代码生成目录内容；目录完全由模板内已有内容决定。因此数字重复问题只能在模板中按上述步骤修改。

若将来改为“由代码生成目录文本”再写入占位符，可在 `goldTemplateData.ts` 或单独模块中生成目录字符串，并在模板中预留如 `{{CONTENTS}}` 占位符。
